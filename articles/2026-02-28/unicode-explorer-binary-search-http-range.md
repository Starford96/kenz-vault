Прототип для пошуку Unicode codepoints, побудований як експеримент із HTTP range-запитами — і як приклад використання LLM для задоволення цікавості.

**Демо:** https://tools.simonwillison.net/unicode-binary-search

## Ідея

Виклик: знайти задачу для бінарного пошуку, де дані природно відсортовані. Claude запропонував пошук метаданих Unicode codepoints — багато МБ відсортованих даних, де бінарний пошук ідеально підходить.

Процес:
1. Claude написав специфікацію для Claude Code
2. Claude Code реалізував код асинхронно проти репозиторію `simonw/research` на GitHub
3. Результат розгорнутий на S3 + Cloudflare, CORS-доступний файл 76,6 МБ

## Ключовий технічний інсайт

**HTTP Range-запити несумісні зі стисненням HTTP.**

Стиснення (gzip/brotli) змінює розміри даних — байтові зміщення у Range-заголовку (`Range: bytes=1000-2000`) більше не відповідають реальному вмісту. Результат — читання неправильного шматка файлу.

Додавати `'Accept-Encoding': 'identity'` не обов'язково: Cloudflare та інші CDN автоматично вимикають стиснення при наявності заголовка `content-range` у відповіді.

## Як це працює

```javascript
// Бінарний пошук через HTTP Range-запити
const response = await fetch(url, {
    headers: { 'Range': `bytes=${start}-${end}` }
});
const chunk = await response.text();
// порівняти chunk з шуканим значенням -> скоригувати діапазон
```

Введіть символ (`ø`) або hex codepoint (`1F99C`) — інструмент покроково покаже кожен крок бінарного пошуку крізь 76 МБ файл.

## Патерн для запозичення

**HTTP Range-запити + бінарний пошук = великі статичні датасети в браузері без бекенду.**

Підходить для будь-яких відсортованих даних: словники, таблиці IP-геолокацій, індекси, логи. Не потрібен сервер — лише S3/Cloudflare і CORS.
